Erreur pour l'envoi d'un email via un serveur mail

Dans le cadre d'un projet web avec le framework CodeIgniter, pour moi il était necessaire d'utiliser l'envoi de mail qui concerne donc le smtp qui utilise le port 25. En local, je n'avais pas de problème pour envoyer des mail, mais une fois cette fonctionnalité déployée sur mon serveur dédié.

ALLO ALLO où es-tu petit mail ???

J'ai pu constater que l'on peut avoir au moins 2 erreurs différentes pour le même problème, voici celles que j'ai trouvé :

  • The following SMTP error was encountered: 111 Connection refused Unable to send data: AUTH LOGIN
  • The following SMTP error was encountered: 530-5.5.1 Authentication Required. / Failed to authenticate password. Error: 535-5.7.8 Username and Password not accepted.

Ma configuration (avant résolution de mon problème) :

J'ai cherché pendant assez longtemp pour finalement découvrir qui suffit de modifier seulement la valeur du protocol. En effet, à la place de smtp, il faut mettre sendmail.
