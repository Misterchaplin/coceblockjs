Test d'égalité

Sur cette page, je vais présenter l'égalité Java avec == et equals.

En effet, beaucoup de personnes se trouvent confortées à ce test, car suivant certains cas l'égalité ne se fait pas comme prévue.

Egalité avec une classe enveloppe

Exemple 1

Commençons d'abord par un exemple avec 2 chaînes String.

String ch1 = "5"; String ch2 = "5"; System.out.println(ch1 == ch2); System.out.println(ch1.equals(ch2));
if("Albert" == "Martin"){ System.out.println("La blague"); }else{ System.out.println("Chapeau"); }

Cet exemple retourne true pour les 2.

Exemple 2

Maintenant voyons un exemple avec instanciation d'un nouveau String :

String ch3 = new String("nouveau"); String ch4 = new String("nouveau"); System.out.println(ch3 == ch4); System.out.println(ch3.equals(ch4);

Cet exemple retourne false pour la première et true pour la deuxième. Le résultat est différent du premier car le fait de créer une nouvelle instance, donne lieu à une nouvelle zone mémoire.

  • == : Retourne false si les 2 variables sont parfaitement identiques
  • equals : Retourne true si les 2 variables ont la même valeur

Egalité avec notre propre classe

Exemple 3

Pour cet exemple, j'ai créé une classe nommée Homme avec pour attribut (age, nom et prénom) ainsi que son constructeur, ses accesseurs et la méthode hashCode() et equals:

if("Albert" == "Martin"){ System.out.println("La blague"); }else{ System.out.println("Chapeau"); } Homme homme1 = new Homme(15, "Doe", "John"); Homme homme2 = new Homme(15, "Doe", "John"); System.out.println(homme1 == homme2); System.out.println(homme1.equals(homme2);

Comme précèdemment nous avons pour retour false pour la première et true pour la suivante.

Au dessus, j'ai dit avoir généré la méthode hashCode et equals pour pouvoir vous montrer le même résultat que dans l'exemple 2.

Désormais, si l'on supprime les 2 méthodes que se passe-t-il ? Eh bien nous avons une réponse false pour les 2. C'est pourquoi, si vous souhaitez avoir le même type de réponse que pour l'exemple 2, il est impératif de générer les 2 méthodes.

Exemple 4

Instance d'une classe VS copie de celle-ci.

Homme homme3 = new Homme(15, "Doe", "John"); Homme homme4 = homme3; System.out.println(homme3 == homme4); System.out.println(homme3.equals(homme4);

Nous avons true pour les 2 car homme4 est la copie parfaite de homme3.

Cette méthode implique que si par exemple nous modifions l'âge de la personne homme4 à 17 au lieu de 15 alors l'âge va aussi valoir 17 pour homme3.

Nous pouvons avoir le même résultat que dans l'exemple 2 si nous faisons un copie en instanciant de nouveau homme3 :

Homme homme4 = new Homme(homme3);

Exemple 5

Une copie avec Cloneable

Pour commencer, il faut implémenter Cloneable sur la classe Homme puis générer la méthode clone().

Homme homme5 = new Homme(15, "Doe", "John"); Homme hommeClonned = (Homme)homme5.clone(); System.out.println(homme3 == hommeClonned); System.out.println(homme3.equals(hommeClonned);

Le résultat est identique à l'exemple 2 (false puis true).

L'object est copié mais l'addresse mémoire est différente.
